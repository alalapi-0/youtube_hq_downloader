# 人工审核反馈闭环

## 为什么要做人审反馈闭环

搜索关键词和自动过滤规则只能粗筛。真正的“适合”通常来自视觉风格、频道可信度、产品聚焦程度、格式质量和业务偏好。人工审核把这些判断结构化成标签，后续统计和 LLM 才能知道哪些搜索策略更有效。

## 人工标签如何影响下一轮搜索

- `pass_features` 高频出现时，会变成下一轮加分方向或关键词扩展。
- `reject_reasons` 高频出现时，会变成 negative keywords 或过滤权重建议。
- 高通过率 `query_used` 会被保留或扩展。
- 低通过率 `query_used` 会被降优先级。
- 多次通过的频道会进入 `channel_whitelist` 建议。
- `not_4k` 高频出现时，规则策略会建议 `require_4k=true`。

## 什么样的数据量适合总结规律

默认配置要求至少 20 条人工审核样本才适合生成比较稳定的策略。少于这个数量时，系统仍会输出统计，但会标记 `sample_size_too_small`，并生成保守策略。

## 如何解释通过率

通过率不是绝对质量评分，而是“在当前搜索词、品牌、频道、过滤规则下，人审认为可用的比例”。通过率高的 query 值得扩展；通过率低的 query 不一定永远无效，可能只是当前样本太小或筛选条件偏差。

## 如何避免过拟合到少量样本

- 不要因为 1 条通过就大幅增加某个关键词。
- 优先观察同时具备高通过率和足够样本数的 query / brand / channel。
- 保留 `uncertain` 状态，不要强行归入 pass/reject。
- LLM 只能总结人工审核结果，不应凭空加入未观察到的品牌或内容类型。
- 对小样本输出只做保守建议，等更多审核数据后再调整硬规则。
